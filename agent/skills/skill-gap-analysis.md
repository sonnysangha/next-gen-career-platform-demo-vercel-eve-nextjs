# Skill-gap analysis

Playbook for turning `analyze_skill_gap` (and `get_relevant_jobs` match data) into
clear, prioritized guidance. Load `get_user_profile` first.

## Read the gap

- Separate the results into three buckets:
  - **Strengths** — required skills the user clearly has. Lead with these.
  - **Partial** — related/adjacent experience that needs framing or a small step up.
  - **Missing** — required skills with no evidence in the profile.
- Tie each skill back to a specific requirement in the target job, not generic advice.

## Prioritize what to learn

- Rank missing skills by: (1) how often they appear across the user's target jobs,
  (2) how much they gate the role (must-have vs nice-to-have), (3) time-to-competence.
- Recommend the 3–5 highest-leverage skills. Resist listing everything.
- For each, suggest the fastest credible path to demonstrable proof (a course, a
  project, a certification).

## Frame honestly

- Give a straight read on fit: strong match, stretch, or not yet — with the reason.
- For stretch roles, show exactly what would close the gap and roughly how long.
- Highlight transferable strengths the match score may undervalue.

## Hand off

- If the user wants to act on the gaps, offer to build a plan with the
  `career-planning` skill, or improve their profile with `profile-optimization`.
- Ground everything in real profile data and real job requirements — never invent
  skills the user hasn't demonstrated.
